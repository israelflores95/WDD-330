Notes

  1. Stack Trace --> a trail to locate the error
  2. Warnings --> The program will still run although it could be unstable
  3. Strict mode --> Catches sloppy programming not just the immediate errors
    You can use "'use strict';" at the top of a javascript page to enable strict mode

      4. Methods for Debugging

    1. the "alert function" --> can help find when code stops running
    2. Using the console --> Find erros via console.log, or within the browser console
    3. Dont forget to remove debugging code when shipping(going into production)
    4. Error objects --> will give an error to a specific reason

      some of the popular ones are..

    1. range error --> number is outside an allowable range
    2. referenceError --> When something is referred to but doesn't exist
    3. SyntaxError --> The way the code is writen is wrong
    4. typeError --> When there is an issue with 2 different data types

      Throwing Exceptions

    1. uses syntax "throw 'something';
    2. use it to cuase an error
    3. best practice throw a specific error(check main.js for more info)
